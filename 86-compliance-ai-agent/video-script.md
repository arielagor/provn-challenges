# Video Walkthrough Script (~3 minutes, ~430 words)

The Compliance team at a mid-sized financial services firm spends 12 to 15 analyst hours a week on first-pass triage of regulatory documents. The job is the same three steps on every document. Extract entities. Classify by domain and urgency. Propose an owner and a due date. My design replaces the first pass with an agent and keeps the human for final approval.

Architecture, in plain English. One supervisor loop, three tools, a fixed plan. The loop ingests a document, chunks it with 500-character overlap so nothing important gets stranded at a chunk boundary, and calls three tools in a fixed order. Extract, classify, propose. Not a free-form ReAct agent, because free-form agents make audit trails impossible. I chose Claude Sonnet as the default model because the workload rewards long context and precise JSON output. The taxonomy and routing table are YAML files a human owns. The agent reads them. The agent never writes them.

Tool schemas. Three tools. Each has a strict JSON schema with required versus optional fields chosen on purpose. For example, the propose_routing tool has a deadline_iso field that is a required string, not an optional nullable. In an earlier draft it was nullable, and the model happily sent back "TBD" as a string value, which broke downstream logic. Explicit empty strings with a clear meaning beat nullable optionals every time with LLMs. Every extracted entity carries a source_span back to the exact characters in the document, so the human reviewer can check the quote without re-reading the whole PDF.

Evaluation. What does "good enough to route to a human" actually look like? Three numbers. Precision on domain classification above 90 percent on a labeled set of 200 historical documents. Recall on hard deadlines above 95 percent, because missing a deadline is worse than inventing one. Median reviewer time per document below 5 minutes, down from roughly 20 minutes today. If any of those three slips, the agent is not ready for live traffic and we hold.

The AI question. The moment I redirected the AI was on architecture. My first prompt asked for an agent design. What came back was the generic tutorial shape. Three separate agents, a vector store, LangGraph orchestration. That is not wrong for a lot of problems. It is wrong for this one. The failure mode here is not lack of knowledge, it is that the important sentence is buried in paragraph 14 of a 40-page PDF. A multi-agent vector-store system does nothing for buried sentences. I threw out the draft and redirected to a single loop with strict JSON tools, paragraph-aware chunking, and a verify-attribution pass on every extracted deadline. The second draft solved the actual problem.

That is the whole design.
